API Monitor – quick capture checklist (Synapse BLE)
===================================================

1. API Monitor should be open (run as Admin). If not:
   Right-click "API Monitor" in Start → Run as administrator.

2. CAPTURE FILTER (left panel)
   - Expand [Kernel32.dll] or [Kernelbase.dll]
   - Expand "File Management" or "I/O" (or "Input/Output")
   - CHECK: DeviceIoControl
   - CHECK (optional): CreateFileW

3. HOOK *ALL* RAZER PROCESSES (we don't know which does BLE)
   - In "Running Processes" find every RazerAppEngine.exe
   - Right-click each → Hook Process (or double-click each)
   - Hook as many as API Monitor allows; note any that fail
   - All hooked ones appear under "Hooked Processes"
   - When you change DPI, the process that shows DeviceIoControl in the log = the one doing BLE

4. CAPTURE
   - Click [Start] (or enable monitoring)
   - In Razer Synapse: change ONE setting (e.g. DPI 800 → 1600)
   - Wait 3–5 seconds
   - Click [Stop]

5. INSPECT
   - In Output/Summary, look for DeviceIoControl rows
   - Note: dwIoControlCode (ioctl), nInBufferSize, and lpInBuffer (hex)
   - Look at "Hex Buffer" or "Parameters" for the input buffer hex

6. SAVE
   - File → Save Capture (e.g. synapse_dpi_capture)
   - For the runbook: write down ioctl (hex), inLen, and payload_hex for any call that looks like a small write (e.g. 8–128 bytes) when you changed DPI
